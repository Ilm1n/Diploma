# Use-Case диаграммы Kantano

В папке лежат PlantUML-исходники диаграмм вариантов использования для дипломного проекта `Kantano`.

## Состав

- [00-overview.puml](./00-main-overview.puml) - общая диаграмма системы для основной части диплома.
- [01-projects-and-team.puml](./01-projects-and-team.puml) - детализация сценариев проектов, команды и приглашений.
- [02-board-and-tasks.puml](./02-board-and-tasks.puml) - детализация сценариев Kanban-доски и задач.

## Подход к оформлению

Диаграммы сделаны по базовым правилам UML use-case:

- акторы названы в единственном числе;
- варианты использования сформулированы как действия пользователя;
- общая диаграмма не перегружена техническими деталями;
- детализация вынесена в отдельные диаграммы по смысловым блокам;
- ролевые права отражены через обобщение акторов: владелец наследует возможности менеджера, менеджер - участника проекта.

## Что не вынесено на диаграммы

В use-case диаграммы не добавлены внутренние технические механизмы:

- refresh token;
- WebSocket heartbeat;
- Redis Pub/Sub;
- внутреннее состояние frontend store;
- генерация API-клиента;
- детали хранения файлов в S3.

Эти элементы относятся к реализации и лучше раскрываются в архитектурной, sequence или deployment-диаграмме.
